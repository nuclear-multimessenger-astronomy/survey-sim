# ZTF GRB Afterglow Detection Efficiency

This example simulates GRB afterglow detection with ZTF over March 2018 -- March 2021 (the ZTFReST analysis period), using the **fiesta** neural network surrogate for relativistic blast wave afterglow emission. We compare predicted detections against the 5 observed afterglows with GRB counterparts reported by ZTFReST.

## Background

GRB afterglows arise from the interaction of a relativistic jet with the circumburst medium, producing broadband synchrotron radiation that fades over days to weeks. The forward shock (FS) dominates at late times, while the reverse shock (RS) can produce a bright optical flash at early times (\(t \lesssim 1\) day).

We use two fiesta surrogates trained on the [blastwave](https://github.com/nuclear-multimessenger-astronomy/blastwave) Rust crate:

| Surrogate | Components | Parameters |
|-----------|------------|------------|
| `blastwave_gaussian_CVAE` | FS only | 8 |
| `blastwave_rs_gaussian_CVAE` | FS + RS | 10 (adds `log10_RB`, `log10_duration`) |

The surrogates predict spectral flux density on a 2D grid (frequencies \(\times\) times), which is then integrated through sncosmo filter transmission curves to produce per-band magnitudes.

## GRB population catalog

The GRB population is drawn from a pre-computed catalog of 42,803 on-axis parameter sets at `/fred/oz480/mcoughli/simulations/argus/GRB_afterglows_argus.csv`. These are generated by a Monte Carlo population synthesis that draws GRB energetics from observed distributions and microphysical parameters from a fitted posterior.

| Parameter | CSV column | Fiesta name | Typical range |
|-----------|-----------|-------------|---------------|
| Isotropic energy | `Eiso` | `log10_E0` \(= \log_{10}[E_\mathrm{iso}(1-\eta)/\eta]\) | \(10^{47}\)--\(10^{54}\) erg |
| Lorentz factor | `Gamma_0` | `log10_lf` | 1--500 |
| Jet half-opening angle | `logthc` | `thetaCore` \(= 10^{\texttt{logthc}}\) | 0.01--0.6 rad |
| Viewing angle | `thv` | `inclination_EM` | 0--\(\theta_c\) (on-axis) |
| ISM density | `logn0` | `log10_n0` | \(10^{-4}\)--\(10^{2}\) cm\(^{-3}\) |
| Electron energy fraction | `logepse` | `log10_epsilon_e` | \(10^{-3}\)--\(10^{0}\) |
| Magnetic energy fraction | `logepsB` | `log10_epsilon_B` | \(10^{-5}\)--\(10^{0}\) |
| Electron spectral index | `p` | `p` | 2.0--3.0 |
| Host extinction | `av` | (applied post-model) | 0--5 mag |

The catalog GRBs are already filtered to on-axis viewing (\(\theta_v \leq \theta_c\) or \(\sin\theta_v \leq 1/\Gamma_0\)), so we use the **on-axis observed rate** rather than the beaming-corrected rate.

## GRB rate

| Rate type | Value | Reference |
|-----------|-------|-----------|
| On-axis (observed) | ~1.3 Gpc\(^{-3}\) yr\(^{-1}\) | Wanderman & Piran 2010 |
| On-axis range | 0.5--2.0 Gpc\(^{-3}\) yr\(^{-1}\) | Literature spread |
| Beaming-corrected | ~800 Gpc\(^{-3}\) yr\(^{-1}\) | Ghirlanda+2022 |

Since the catalog contains only on-axis GRBs, we adopt 1.3 Gpc\(^{-3}\) yr\(^{-1}\) as the fiducial rate.

import survey_sim.gpu_setup  # configure CUDA for JAX

## Load the ZTF survey

```python
survey = load_ztf_survey(start="201803", end="202103", nside=64)
# ~504,000 observations, 3.09 yr, bands: g, r, i
```

Survey data is automatically downloaded from HuggingFace on first use.

## Detection criteria

We adopt ZTFReST-like criteria for fast optical transients:

```python
det = DetectionCriteria(
    snr_threshold=5.0,              # primary detection at 5 sigma
    snr_threshold_secondary=3.0,    # secondary at 3 sigma
    min_detections=2,               # at least 2 detections
    min_detections_primary=1,       # at least 1 at full depth
    max_timespan_days=14.0,         # within 14-day window
    min_time_separation_hours=3.0,  # >=3h between detections
    require_fast_transient=True,    # must show rapid evolution
    min_rise_rate=0.0,              # no rise rate cut
    min_fade_rate=0.3,              # >=0.3 mag/day fading
    min_galactic_lat=15.0,          # exclude galactic plane
)
```

## Load the afterglow model

### Forward shock only

```python
model = FiestaAfterglowModel(
    name="blastwave_gaussian_CVAE",
    band_map=ZTF_BAND_MAP,
)
```

### Forward + reverse shock

```python
model = FiestaAfterglowModel(
    name="blastwave_rs_gaussian_CVAE",
    band_map=ZTF_BAND_MAP,
)
```

The `FiestaAfterglowModel` adapter handles:

- **Parameter mapping**: converting CSV catalog columns to fiesta surrogate parameter names (e.g., `Eiso` \(\to\) `log10_E0` with gamma-ray efficiency correction)
- **Parameter clamping**: clipping values to the surrogate's valid training range
- **Batch evaluation**: GPU-accelerated JAX vmap in chunks of 4096 transients
- **Temporal interpolation**: interpolating the surrogate's time grid to actual observation times

## Run the simulation

```python
grb_pop = GrbPopulation(
    "GRB_afterglows_argus.csv",
    rate=1.3,      # Gpc^-3 yr^-1 (on-axis)
    z_max=6.0,
)

pipe = SimulationPipeline(
    survey, [grb_pop], {"Afterglow": model}, det,
    n_transients=100_000, seed=42,
)
result = pipe.run()
```

## Results

### Forward shock only

| Quantity | Value |
|----------|-------|
| Transients simulated | 100,000 |
| Spatially matched | 73,712 |
| Detected | 112 |
| Efficiency | 0.112% |
| Elapsed | ~35s |

### Forward + reverse shock

| Quantity | Value |
|----------|-------|
| Transients simulated | 100,000 |
| Spatially matched | 73,712 |
| Detected | 226 |
| Efficiency | 0.226% |
| Elapsed | ~35s |

The reverse shock doubles the detection efficiency by brightening early-time emission, making more afterglows accessible to ZTF's rapid cadence.

### Expected detections vs. observations

Using the on-axis rate and the FS+RS model:

| Rate (Gpc\(^{-3}\) yr\(^{-1}\)) | Expected detections |
|----------------------------------|---------------------|
| 0.5 | 1.9 |
| 1.0 | 3.7 |
| **1.3** | **4.9** |
| 2.0 | 7.5 |

ZTFReST reported **5 afterglows with GRB counterparts** over the same period. The FS+RS model at the fiducial on-axis rate of 1.3 Gpc\(^{-3}\) yr\(^{-1}\) predicts **4.9 detections** --- excellent agreement.

The FS-only model predicts ~2.4 at the same rate, underestimating by a factor of ~2, highlighting the importance of the reverse shock for early-time detection.

!!! note "Orphan afterglows"
    ZTFReST also found 2 orphan afterglows (no GRB trigger). At a rate of 2.0 Gpc\(^{-3}\) yr\(^{-1}\), the FS+RS model predicts 7.5 detections, consistent with the total of 7 (5 triggered + 2 orphan).

### Rate upper limits

```python
import math

eff = result.n_detected / result.n_simulated
V_eff = 1140.0 * 0.47  # Gpc^3 (z < 6, ZTF sky fraction)
VT = V_eff * survey.duration_years * eff

R90 = -math.log(0.10) / VT
R95 = -math.log(0.05) / VT
```

| Model | Efficiency | \(R_\mathrm{upper}\) (90% CL) | \(R_\mathrm{upper}\) (95% CL) |
|-------|-----------|-------------------------------|-------------------------------|
| FS only | 0.112% | 1 Gpc\(^{-3}\) yr\(^{-1}\) | 2 Gpc\(^{-3}\) yr\(^{-1}\) |
| FS + RS | 0.226% | 1 Gpc\(^{-3}\) yr\(^{-1}\) | 1 Gpc\(^{-3}\) yr\(^{-1}\) |

!!! tip "GPU acceleration"
    The fiesta surrogate uses JAX for batch evaluation. With `import survey_sim.gpu_setup` before loading the model, JAX will use CUDA if available. 100K transients complete in ~35 seconds on GPU. The `batch_evaluate_arrays` method processes transients in chunks of 4096 to avoid GPU memory exhaustion.

## Full script

```python
#!/usr/bin/env python
"""ZTF GRB afterglow detection efficiency with fiesta surrogate (FS+RS)."""
import sys
sys.path.insert(0, "/path/to/survey-sim/python")

import survey_sim.gpu_setup  # noqa: F401

import math
import time

from survey_sim import (
    GrbPopulation,
    DetectionCriteria,
    SimulationPipeline,
    load_ztf_survey,
)
from survey_sim.fiesta_afterglow_model import FiestaAfterglowModel, ZTF_BAND_MAP

# Survey
survey = load_ztf_survey(start="201803", end="202103", nside=64)

# Detection criteria (ZTFReST-like)
det = DetectionCriteria(
    snr_threshold=5.0, snr_threshold_secondary=3.0,
    min_detections=2, min_detections_primary=1,
    max_timespan_days=14.0, min_time_separation_hours=3.0,
    require_fast_transient=True, min_rise_rate=0.0, min_fade_rate=0.3,
    min_galactic_lat=15.0,
)

# Model (FS + RS)
model = FiestaAfterglowModel(
    name="blastwave_rs_gaussian_CVAE",
    band_map=ZTF_BAND_MAP,
)

# Population (on-axis rate)
grb_pop = GrbPopulation(
    "GRB_afterglows_argus.csv",
    rate=1.3,      # Gpc^-3 yr^-1
    z_max=6.0,
)

# Run
pipe = SimulationPipeline(
    survey, [grb_pop], {"Afterglow": model}, det,
    n_transients=100_000, seed=42,
)
result = pipe.run()

# Results
eff = result.n_detected / max(result.n_simulated, 1)
print(f"Detected: {result.n_detected} / {result.n_simulated} = {eff:.4%}")

V_eff = 1140.0 * 0.47
VT = V_eff * survey.duration_years * eff
R90 = -math.log(0.10) / VT
print(f"R_upper (90% CL) = {R90:.0f} Gpc^-3 yr^-1")

# Expected detections
for rate in [0.5, 1.0, 1.3, 2.0]:
    expected = rate * V_eff * survey.duration_years * eff
    print(f"  Rate {rate:.1f}: {expected:.1f} expected")
```
